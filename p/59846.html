将Blog部署到Vercel

最近还是在想办法优化网站的访问速度，然后发现了一个不错的静态托管平台：Vercel。

能将blog托管在上面，且速度比gitpage更快。

大概理解，Vercel提供静态资源的托管服务，所以我们可以把托管在Github上的blog转移到Vercel上。Vercel在中国也有服务器，所以可以获得更快的访问速度。Vercel也支持自定义域名，而且完全免费。

第一步要有一个Github仓库，也就是blog的仓库。

第二步注册一个Vercel账号。注册时可以直接选用Github登录，很方便。

第三步将Github仓库导入Vercel。首先在主页（Hobby）选择 New Project 。然后会看到如图界面。

我这里已经导入了仓库，如果是第一次使用，应该先选择左侧的下拉栏，选择 Add Github Account 增加一个账户，然后按照提示给需要部署的仓库安装Vercel，然后将其导入即可。

PROJECT NAME 一栏填入项目名，与默认提供的系统域名有关。

这里要注意 FRAMEWORK PRESET 一栏，它提供了多种框架供你选择。我这里默认的是 Other ，我之前试过使用 Hexo 的框架，发现blog无法正常加载，而使用默认的 Other 就好了(。总之系统会自动识别你的仓库内容，然后选择框架，如果默认的框架没有生效，可以再自己选择对应的框架。

然后部署，成功后应该是这个界面:

系统会提供一个 xxx.vercel.app 的自动域名，前面的部分就是你的 PROJECT NAME 。然后你可以点击 View Domains 自定义域名。

然后去你购买域名的地方设置DNS记录，系统提供的是默认的DNS记录。

速度也是很可观的。

多次平均后，Github的全部节点访问速度为1.787s，Vercel的全部节点访问速度为0.934s

平均下来Vercel速度更快，不过在华东地区反而GitHub会更快一些。
